建立网站的过程

起初，我通过自己的 GitHub 账户创建了静态网页仓库，并用最基本的 index.html 页面作为网站主页。这是我第一次从零开始搭建个人站点，起初所有内容都是手动嵌入的，导航栏和脚注每页都需要单独复制粘贴，维护起来非常繁琐。

随后，我对项目结构进行了优化：将导航栏 navbar.html 和脚注 footer.html 抽离成单独文件，通过 navbar.js 脚本在页面加载时动态插入，实现了模块化。现在只需修改一个文件，整个站点就会统一更新，极大地提升了维护效率。

网站样式通过 style.css 管理，保证了内容展示的一致性和扩展性。每新增一个页面，我只需关注内容本身，而不必重复搭建结构。

未来，我希望将这个流程整理成一个简单的“建站接口”或脚手架，结合我的公众号“离谱纪”，为关注者提供一键生成网站的服务，让更多人能轻松拥有自己的技术主页。

虽然我最初希望这个网站承载更多技术输出，但现在我更重视“先做出东西”。因为我还有很多想法在路上，网站本身就是我“先实现再分享”的实验场。

项目源代码及更新计划可见我的 GitHub 仓库。
